# Implementation Plan: Web Crawler

**Branch**: `027-web-crawler` | **Date**: 2026-03-17 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/027-web-crawler/spec.md`

## Summary

A web crawler MFE already exists (PR #774) with multi-page spidering, job
management, document collection, and trace logging. This plan addresses the
**gaps** between the existing implementation and the spec: keyword search of
crawl history (FR-011), SSRF protection for the server-side fetcher (FR-015),
richer metadata extraction (FR-003: author, publish date, description, images),
and a content reader view (FR-004/FR-009). All data operations use the existing
GraphQL service — no REST endpoints.

## Technical Context

**Language/Version**: TypeScript 5.x (frontend + Cloud Functions)
**Primary Dependencies**: React 18, Apollo Client/Server, cheerio (HTML parsing), axios (HTTP fetching)
**Storage**: Firestore (`users/{uid}/crawlJobs/{jobId}`, subcollections: `documents`, `traces`)
**Testing**: Vitest + React Testing Library (frontend), `cd functions && npx tsc --noEmit` (backend)
**Target Platform**: Web (desktop + mobile responsive), Firebase Cloud Functions (Node.js runtime)
**Project Type**: Web application (MFE in pnpm monorepo)
**Performance Goals**: Crawl completes within 15 seconds for single pages (SC-001)
**Constraints**: Firestore 1MB document limit (content must be truncated), 540s Cloud Function timeout
**Scale/Scope**: Single user's private crawl history, up to 50 jobs displayed, 200 traces per job
