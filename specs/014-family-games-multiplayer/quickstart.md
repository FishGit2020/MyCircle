## Prerequisites

```bash
# Ensure you are on the feature branch
git checkout 014-family-games-multiplayer

# Install and build shared
pnpm install
pnpm build:shared
```

## Running the family-games MFE in dev

```bash
# Start shell + family-games together
pnpm dev:family-games

# Or start all MFEs (slower)
pnpm dev
```

Navigate to `http://localhost:3000/family-games` to see the existing game hub.

## Implementation Order

Follow the task order in `tasks.md` (generated by `/speckit.tasks`). The dependency chain is:

```
StorageKeys → hooks → data files → components → FamilyGames integration → i18n → tests
```

Do **not** start components before the hooks are in place — TournamentSetup, Handoff, and Results all depend on `useTournament`.

## Architecture Notes

- **Tournament state machine**: `useTournament` persists to `localStorage[FAMILY_GAMES_TOURNAMENT]`. The machine advances through `{ round, playerIndex, gameIndex }` tuples.
- **Turn handoff**: `FamilyGames.tsx` switches between `TournamentHandoff` (covers screen) and the actual game component. After the game calls `onGameEnd(score, timeMs)`, the hook records the score and navigates to the next handoff.
- **Beat the Clock**: Reuses the `Timer` component. Prompts are pre-shuffled. "Confirmer" taps the large button; the same device is used for both talker and confirmer.
- **Sequence difficulty**: Easy = only arithmetic puzzles (`type === 0`); Medium = existing `Math.floor(Math.random() * 4)`; Hard = all 4 types + a 15-second per-puzzle `Timer`.
- **Scores + auth**: `window.__familyGames.saveScore` requires auth. If user is not logged in, skip the Firestore save but still update the local `PlayerProfile.personalBests`. Show a silent note ("Sign in to save to leaderboard") rather than blocking the flow.
