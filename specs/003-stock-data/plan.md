# Implementation Plan: Candlestick Charts + Unified Refresh for Stock Tracker

**Branch**: `003-stock-data` | **Date**: 2026-03-18 | **Spec**: specs/003-stock-data/spec.md
**Input**: Feature specification from `/specs/003-stock-data/spec.md`

## Summary

Enhance the existing `StockTracker` MFE with two targeted improvements: (1) add a candlestick chart mode to `StockChart.tsx` — the OHLC data is already fetched; this is purely a rendering enhancement; (2) unify the refresh experience so the existing refresh button also re-fetches candles, and add a refresh button to `StockNews.tsx` (switching from `cache-first` to `cache-and-network`). No polling is introduced — `pollInterval=0` is enforced and preserved.

No new MFE packages, no GraphQL schema changes, no new resolvers. Changes touch 3 files in `stock-tracker` and i18n in `shared`.

## Implementation Notes

### US1: Candlestick chart mode in `StockChart.tsx`

1. Add `const [chartType, setChartType] = useState<'line' | 'candlestick'>('line')` at the top of `StockChart`
2. In the chart header, alongside the timeframe buttons, add a two-button pill: `[Line] [Candle]` using `t('stocks.chartLine')` / `t('stocks.chartCandle')`
3. Candlestick render path (replaces the line/area path block when `chartType === 'candlestick'`):
   - Compute `barWidth = plotWidth / Math.max(1, candles.c.length - 1) * 0.6`
   - For each index `i`:
     - `bodyTop = Math.min(getY(o[i]), getY(c[i]))`
     - `bodyBottom = Math.max(getY(o[i]), getY(c[i]))`
     - `bodyHeight = Math.max(1, bodyBottom - bodyTop)` — clamp to 1px so flat candles are visible
     - `isUp = c[i] >= o[i]`
     - `color = isUp ? '#22c55e' : '#ef4444'`
     - Render: upper wick `<line>`, body `<rect>`, lower wick `<line>`
4. Keep existing min/max annotations (using close prices for consistency across both modes)
5. Keep existing area gradient for line mode; hide it in candlestick mode
6. The SVG `aria-label` should reflect chart type: `{symbol} ${chartType === 'candlestick' ? 'candlestick' : 'price'} chart`

### US2: Unified refresh in `StockTracker.tsx`

Current (line 46):
```ts
const { candles: selectedCandles, loading: candlesLoading } = useStockCandles(selectedSymbol, timeframe);
```

Change to:
```ts
const { candles: selectedCandles, loading: candlesLoading, refetch: candlesRefetch } = useStockCandles(selectedSymbol, timeframe);
```

Refresh button `onClick`:
```ts
onClick={() => { quoteRefetch(); candlesRefetch(); }}
```

### US2: News refresh in `StockNews.tsx`

1. Change `fetchPolicy: 'cache-first'` → `fetchPolicy: 'cache-and-network'`
2. Destructure `refetch: newsRefetch` from the `useQuery` call
3. Add small `<button type="button">` with `aria-label={t('stocks.refreshNews')}` in the news section header, calling `newsRefetch()` on click, disabled while `loading`
