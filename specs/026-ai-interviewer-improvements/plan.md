# Implementation Plan: AI Interviewer MFE Improvements

**Branch**: `026-ai-interviewer-improvements` | **Date**: 2026-04-07 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/026-ai-interviewer-improvements/spec.md`

## Summary

Improve the existing AI Interviewer MFE by adding a session history view with search/filter, a cross-session progress analytics panel, bulk question bank import/export, an optional countdown timer, and filling the test coverage gaps for three untested components. The backend approach extends the existing Firestore session metadata document to store score summaries and chapter/difficulty, enabling efficient listing and analytics without N+1 Cloud Storage reads. All new data operations go through the existing GraphQL service.

## Technical Context

**Language/Version**: TypeScript 5.x (strict mode)
**Primary Dependencies**: React 18, @mycircle/shared (Apollo Client, i18n, GraphQL), Tailwind CSS, Vite Module Federation, Firebase Cloud Functions, Firestore, Cloud Storage
**Storage**: Firestore (session metadata) + Cloud Storage (full session JSON) — existing dual-tier architecture
**Testing**: Vitest, React Testing Library — existing per-package config
**Target Platform**: Web (Module Federation MFE), mobile-first responsive
**Project Type**: Micro-frontend (MFE) improvement — no new package scaffolding
**Performance Goals**: Session history list renders in under 1s for up to 100 sessions; progress analytics chart renders from pre-computed Firestore fields (no Cloud Storage reads at list time)
**Constraints**: All imports from `@mycircle/shared`, never `@apollo/client` directly; every color class needs `dark:` variant; all strings via `t('key')` in 3 locales; touch targets ≥ 44px; no `100vh` calculations in MFE pages
**Scale/Scope**: Single MFE package; ~5 new/modified components; ~3 new test files; 1 Firestore index addition; 5 GraphQL schema changes
