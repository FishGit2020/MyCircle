# Implementation Plan: Weather Data for Saved Cities

**Branch**: `002-weather-data` | **Date**: 2026-03-18 | **Spec**: specs/002-weather-data/spec.md
**Input**: Feature specification from `/specs/002-weather-data/spec.md`

## Summary

Enrich the weather experience for saved/favorite cities with three targeted enhancements: (1) add AQI badge, forecast high/low, and historical note to favorite city cards on the landing page; (2) add a city switcher chip row to the weather detail page shell wrapper; (3) create `WeatherWrapper` to pass `favoriteCities` from AuthContext into `WeatherDisplay` MFE so the inline comparison panel uses real saved cities instead of a sessionStorage hack.

All GraphQL queries already exist. No new resolvers, no new MFE packages, no schema changes. Implementation touches three existing packages: `shell`, `weather-display`, and `shared` (i18n only).

## Implementation Notes

### US1: FavoriteCityCard enhancements

`FavoriteCityCard` in `FavoriteCities.tsx` already calls `GET_CURRENT_WEATHER`. Add two more parallel queries:
- `useQuery(GET_AIR_QUALITY, { variables: { lat, lon }, fetchPolicy: 'cache-first' })` → render AQI badge
- `useQuery(GET_FORECAST, { variables: { lat, lon }, fetchPolicy: 'cache-first' })` → render `forecast[0].temp.{min,max}`
- `useQuery(GET_HISTORICAL_WEATHER, { variables: { lat, lon, date }, fetchPolicy: 'cache-first' })` → render historical note if `|current.temp - historical.temp_max| >= 3`

AQI badge color mapping from data-model.md (5 levels, dark mode variants required).

Skeleton placeholders during loading (match existing pattern in the card).

### US2: City switcher in shell WeatherPage

Check if a `WeatherPage` component exists in `App.tsx` or as a separate file. If inline, extract a `CitySwitcher` component. Read `favoriteCities` from `useAuth()`, parse current city from `useParams()` (`coords` = `lat,lon`). Filter out current city. Render horizontal scrollable `<div>` of `<Link>` chips. Hidden when `otherFavorites.length === 0`.

### US3: WeatherWrapper + prop injection

1. Create `packages/shell/src/components/widgets/WeatherWrapper.tsx` (mirrors TransitWrapper)
2. Update `packages/shell/src/remotes.d.ts` to declare `WeatherDisplay` props
3. Update `App.tsx` to use `<WeatherWrapper />` instead of direct `<WeatherDisplayMF />` mount
4. Update `WeatherDisplay.tsx` to accept optional `favoriteCities?: FavoriteCity[]` and `currentCityId?: string`
5. Replace `getRecentCitiesFromStorage()` call with the prop (fall back to sessionStorage when prop is undefined, for backward-compat with unauthenticated users)
