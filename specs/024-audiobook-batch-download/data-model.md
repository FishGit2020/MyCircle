# Data Model: Audiobook Batch Download

## Entities

---

### 1. Book (extended)

**Firestore path**: `books/{bookId}`

Existing document extended with 5 new fields:

| Field | Type | Default | Notes |
|-------|------|---------|-------|
| `zipStatus` | `string` | `'none'` | One of: `'none'` \| `'processing'` \| `'ready'` \| `'error'` |
| `zipUrl` | `string \| null` | `null` | Firebase Storage authenticated download URL |
| `zipSize` | `number \| null` | `null` | ZIP file size in bytes |
| `zipGeneratedAt` | `string \| null` | `null` | ISO 8601 timestamp of when ZIP was completed |
| `zipError` | `string \| null` | `null` | Error message when `zipStatus === 'error'` |

**Existing fields** (unchanged): `id`, `title`, `author`, `description`, `language`, `coverUrl`, `epubUrl`, `fileSize`, `chapterCount`, `totalCharacters`, `uploadedBy`, `uploadedAt`, `audioStatus`, `audioProgress`, `audioError`

**State transitions for `zipStatus`**:
```
none ──→ processing ──→ ready
                  ╰──→ error ──→ processing (retry)
ready ──→ processing (regenerate)
```

---

### 2. ZIP Job

**Firestore path**: `books/{bookId}/zipJobs/{jobId}`

Ephemeral trigger document. Created when `requestBookZip` mutation fires, consumed and updated by the `onZipJobCreated` Cloud Function.

| Field | Type | Notes |
|-------|------|-------|
| `bookId` | `string` | Matches parent document ID |
| `status` | `string` | `'pending'` → `'processing'` → `'complete'` \| `'error'` |
| `createdAt` | `Timestamp` | Server timestamp at creation |
| `updatedAt` | `Timestamp` | Last status update |
| `error` | `string \| null` | Error message if `status === 'error'` |

**Lifecycle**: The job document is created by the GraphQL resolver and deleted as part of `deleteBookZip` / `permanentDeleteBook` cleanup. It is NOT deleted by the worker on completion — kept for audit purposes.

---

### 3. Chapter Audio File (unchanged)

**Firestore path**: `books/{bookId}/chapters/{chapterId}`

Relevant fields used by the download feature:

| Field | Type | Notes |
|-------|------|-------|
| `index` | `number` | 0-based chapter index |
| `title` | `string` | Human-readable chapter title |
| `audioUrl` | `string \| undefined` | Firebase Storage download URL; absent if not converted |

---

### 4. ZIP Archive File (Storage)

**Firebase Storage path**: `books/{bookId}/audiobook.zip`

| Property | Value |
|----------|-------|
| Content-Type | `application/zip` |
| Cache-Control | `private, no-cache` (ZIP contains authenticated-token URLs) |
| Download token | Generated by `uploadToStorage()`, embedded in `zipUrl` |
| Naming inside ZIP | `{Book Title} - Ch{index} {Chapter Title}.mp3` per entry |

  # ... existing mutations ...

---

## Validation Rules

- `requestBookZip`: book must exist, not soft-deleted, have at least 1 chapter with `audioUrl`, and `zipStatus !== 'processing'`
- `deleteBookZip`: book must exist; no chapter/audio constraints
- `onZipJobCreated` worker: must verify job `status === 'pending'` before proceeding (idempotency guard against duplicate triggers)

---

## Cleanup Rules

| Trigger | Cleanup Action |
|---------|----------------|
| `permanentDeleteBook` | Delete `books/{bookId}/audiobook.zip` from Storage + all docs in `books/{bookId}/zipJobs` subcollection |
| `deleteBookZip` mutation | Delete `books/{bookId}/audiobook.zip` from Storage + all docs in `books/{bookId}/zipJobs` + reset book zip fields |
| `requestBookZip` (new zip while old exists) | Old zip file is overwritten by the new upload; `deleteBookZip` is NOT called first — the worker writes over the existing path |
